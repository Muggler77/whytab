# whytab 0.5.4

## 中文

0.5.4 集中修复账号切换、云同步边界和本机数据安全问题。数据结构版本仍为 1，更新不会删除、重建或重新分配已有用户数据。

### 数据与账号安全

- 登录、退出、自动同步和手动同步使用账号操作代次校验。旧账号的异步任务在退出或切换账号后会立即失效，不能再覆盖当前界面或本机账号数据。
- 同步结果先写入对应账号的 IndexedDB，并再次确认账号未变化，最后才更新界面。
- 本机上传的快捷图标、文件夹图标、照片、照片文件名和自定义壁纸不会上传到云端。
- 拉取或合并云端数据时，当前设备的本机照片、壁纸和上传图标会被保留。
- 本机存储写入失败时会显示明确提醒，不再静默忽略。

### 云端边界

- 客户端和 Supabase 数据库函数都限制单个云端快照不超过 2 MB。
- 数据库只接受已登录用户写入固定的 `primary` 快照，并拒绝空载荷、非法修订号和其他快照名称。
- 旧细粒度同步表不再向网页或扩展客户端开放直接访问；快照表只读，写入统一通过带修订号检查的原子 RPC 完成。
- 外部网站图标缓存最多保留 200 项，避免长期使用造成无上限缓存增长。

### 发布可靠性

- Cloudflare Pages 自动部署必须配置 `CLOUDFLARE_API_TOKEN` 和 `CLOUDFLARE_ACCOUNT_ID`。
- 任一 Secret 缺失时部署任务会明确失败，不再跳过部署后仍显示成功。
- Chrome/Edge 扩展与 Cloudflare Pages 继续使用独立输出目录；扩展包不包含 `_headers`。

最低同步兼容版本仍为 0.5.2。

## English

Version 0.5.4 hardens account switching, cloud-sync boundaries, and local data handling. The data schema remains version 1, so the update does not delete, rebuild, or reassign existing user data.

### Data and account safety

- Login, logout, automatic sync, and manual sync now use an account-operation generation guard. In-flight work from an old account becomes invalid immediately after logout or account switching and cannot overwrite the active UI or local account data.
- Sync results are persisted to the intended account in IndexedDB, revalidated, and only then applied to the UI.
- Uploaded shortcut icons, folder icons, photos, photo filenames, and custom wallpapers remain device-local and are removed from cloud snapshots.
- Pull and merge operations preserve the current device's local photos, wallpapers, and uploaded icons.
- Local persistence failures now produce a visible warning instead of being silently ignored.

### Cloud boundaries

- Both the client and Supabase database function enforce a 2 MB maximum cloud snapshot size.
- The database accepts only authenticated writes to the fixed `primary` snapshot and rejects null payloads, invalid revisions, and alternate snapshot names.
- Legacy record tables are no longer directly accessible to web or extension clients. The snapshot table is read-only and all writes use an authenticated atomic RPC with revision checks.
- The external website-icon cache is capped at 200 entries to prevent unbounded growth.

### Release reliability

- Cloudflare Pages deployment requires both `CLOUDFLARE_API_TOKEN` and `CLOUDFLARE_ACCOUNT_ID`.
- Missing Secrets now fail the deployment job instead of silently skipping deployment and reporting success.
- Chrome/Edge and Cloudflare Pages continue to use separate output folders; `_headers` is never included in the extension package.

The minimum sync-compatible version remains 0.5.2.
