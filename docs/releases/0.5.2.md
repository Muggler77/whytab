# whytab 0.5.2

## 中文

0.5.2 是一次数据安全、同步可靠性和交互稳定性更新。

- 同步前恢复点和更新前备份改为账号隔离，恢复时校验当前账号，避免同一浏览器内不同账号交叉恢复。
- 普通退出登录只结束当前设备会话，不再影响同一账号的其他设备。
- 云端快照增加服务器维护的 revision，并通过原子写入函数实现乐观并发控制；直接表写入被禁用，冲突时重新拉取、合并并重试。
- 快捷方式与文件夹重新排序时，为所有顺序变化的项目更新时间，防止其他设备恢复旧顺序。
- “导出备份”升级为完整 AppState 备份，恢复时保留当前设备、登录状态和同步服务配置。
- 私人照片和自定义壁纸保持本地，不进入云端快照；自定义壁纸限制为最多 12 张并压缩存储。
- 离线启动优先使用本机缓存会话，账号数据加载完成前不切换本地数据分区。
- 新增密码重置和已登录状态下修改密码入口，注册密码最低为 10 个字符。
- 图标解析结果持久缓存，只预加载首屏少量图标，其余按需加载；32px 以上正常 favicon 不再被错误拒绝。
- 设置窗口使用固定标题栏和独立滚动区，版本信息提高可读性；自动隐藏导航使用稳定边缘触发区和延迟收起。
- Web 版增加 CSP、HSTS、Permissions-Policy、完整 App Shell 预缓存和 Cloudflare Pages 自动部署。

数据结构版本继续保持为 1。数据库新增同步 revision 字段和原子同步函数，旧快照自动获得初始 revision，不删除或重建用户数据。

由于旧客户端不支持原子同步，最低支持版本提升为 0.5.2。旧版本仍可读取本地数据和导出备份，但云端写入会被安全拒绝，升级后恢复正常同步。

## English

Version 0.5.2 focuses on data safety, sync reliability, and interaction stability.

- Restore points and migration backups are scoped to the active account and verified before restore.
- Normal sign-out now ends only the current device session.
- Cloud snapshots use a server-maintained revision and an atomic optimistic-concurrency function. Direct table writes are disabled, with pull, merge, and retry on conflicts.
- Reordering updates every record whose order changed, preventing stale order values from returning on another device.
- Export now creates a complete AppState backup while restore preserves the current device, session, and hosted service configuration.
- Private photos and custom wallpapers remain device-local and are excluded from cloud snapshots. Custom wallpapers are compressed and capped at 12.
- Offline startup uses the cached session, and account storage is not switched until account data loads successfully.
- Password reset and signed-in password update flows are available, with a 10-character minimum for registration.
- Icon resolution is persisted, only a small first viewport set is preloaded, remaining icons are lazy-loaded, and normal 32px favicons are accepted.
- The settings dialog now has a fixed header and reliable scroll region. Auto-hidden navigation uses a stable edge target and delayed close behavior.
- The hosted app adds CSP, HSTS, Permissions-Policy, complete app-shell precaching, and automated Cloudflare Pages deployment.

The local data schema remains version 1. The database migration adds a revision column and atomic sync function without deleting or rebuilding user data.

Because older clients do not support atomic sync, the minimum supported version is now 0.5.2. Older builds retain local access and export capability, but cloud writes are safely rejected until the client is updated.
