# whytab 0.4.1

## 中文

0.4.1 是一次图标清晰度与右键交互可靠性修复。它不修改数据结构、注册登录、账号隔离、同步协议或本地存储逻辑。

### 图标

- 快捷图标优先使用维护过的矢量品牌图和 256px 图标源，再尝试网站提供的高清图标。
- 位图在显示前会检查原始尺寸；低于清晰度要求的小图不会被强行放大。
- 图标正在加载、加载失败或第三方图标源不可用时，会保留清晰的文字占位，不再显示空白区域。
- 移除可能让位图变软的永久缩放与滤镜，并为重复图标复用已验证的结果。
- 文件夹自定义图标使用同一套清晰度检查和失败降级策略。

### 右键菜单

- 主页、网站页、Dock、文件夹弹层和小组件统一由应用根层识别右键目标。
- 修复 Chrome 窗口短暂失焦时菜单立即关闭的问题。
- 修复页面时钟刷新导致菜单监听器重复注册、菜单状态不稳定的问题。
- 菜单仍可通过点击外部、按 Escape 或调整窗口大小正常关闭。

### 数据安全

- 数据结构版本仍为 1，没有新增迁移步骤。
- 本地 IndexedDB、账号数据隔离和云同步行为保持不变。
- 更新不会清除、覆盖或重新归属已有快捷方式、小组件、笔记、待办和设置。

## English

Version 0.4.1 fixes icon sharpness and context-menu reliability. It does not change the data schema, registration or sign-in flow, account isolation, sync protocol, or local-storage behavior.

### Icons

- Shortcut icons now prefer curated vector brand marks and 256px icon sources before trying high-resolution assets published by each site.
- Raster dimensions are checked before display, so undersized images are never enlarged into visibly blurred icons.
- A crisp text fallback remains visible while an icon loads or whenever every external source fails.
- Permanent image scaling and filters that could soften raster artwork have been removed, and verified results are reused for duplicate icons.
- Custom folder artwork uses the same resolution checks and fallback behavior.

### Context menus

- The home page, website page, dock, folder view, and widgets now share one root-level context-target handler.
- Menus no longer close when Chrome briefly reports that the window lost focus.
- Stable listeners prevent clock updates from repeatedly attaching and removing menu handlers.
- Clicking outside, pressing Escape, or resizing the window still closes a menu normally.

### Data safety

- The data schema remains at version 1 and requires no new migration.
- Local IndexedDB storage, account isolation, and cloud-sync behavior are unchanged.
- Updating does not clear, overwrite, or reassign existing shortcuts, widgets, notes, todos, or settings.
