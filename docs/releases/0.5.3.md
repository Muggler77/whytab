# whytab 0.5.3

## 中文

0.5.3 修复 Chrome/Edge 扩展发布包误包含 Cloudflare Pages 专用 `_headers` 文件的问题。Chrome 会拒绝加载以下划线开头的保留文件，因此 0.5.2 的本地扩展目录可能显示“无法加载清单”。

- 浏览器扩展继续输出到 `extension/dist`，其中不再包含任何 Cloudflare 专用文件。
- Web 版本改为独立输出到 `extension/web-dist`，仅该目录包含 `_headers`。
- GitHub 扩展发布包和 Cloudflare 自动部署分别使用对应目录，避免同类问题再次出现。
- 不修改用户数据结构、账号、同步逻辑或云端数据。

最低同步兼容版本仍为 0.5.2，0.5.3 仅修复发布打包。

## English

Version 0.5.3 fixes a packaging issue where the Chrome/Edge extension bundle incorrectly included Cloudflare Pages' `_headers` file. Chromium rejects reserved filenames beginning with an underscore, causing the 0.5.2 unpacked extension to fail during loading.

- Browser extensions continue to build into `extension/dist`, which no longer contains Cloudflare-only files.
- The web app now has a separate `extension/web-dist` output containing `_headers`.
- GitHub extension releases and Cloudflare deployments use their respective build directories.
- No user data schema, account, sync behavior, or cloud data is changed.

The minimum sync-compatible version remains 0.5.2. Version 0.5.3 is a packaging-only correction.
