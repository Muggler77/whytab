# whytab 0.5.6

## 中文

0.5.6 继续收紧多设备同步和公开产品的数据安全边界。数据结构版本仍为 1；现有账号、本机数据和云端快照不会被重建或清空。

- 设置改为按字段合并；小组件开关、组件尺寸和日历记录进一步按内部条目合并，降低不同设备同时修改不同设置时的数据覆盖风险。
- 自定义导航页面删除改为同步删除标记，离线旧设备重新上线后不会恢复已删除页面。
- 快捷地址统一限制为 HTTP/HTTPS，导入、完整备份和历史云端数据中的危险协议不会被打开。
- Chrome/Edge 扩展通过 Chrome Search API 使用用户的默认搜索引擎。
- 图标候选源调整为公共缓存优先并缩短失败超时，同时保留懒加载、文字占位和有界缓存。
- 增加账号永久删除入口。用户必须重新输入当前邮箱和密码，删除后 Auth 账号、云端同步数据和当前设备上的账号数据会一并清理。
- 最低可同步版本提升至 0.5.6，阻止旧客户端继续使用整块设置覆盖算法写入新快照。
- 普通用户安装说明改为直接下载 GitHub Release 构建产物。

本版本不启用端到端加密。本机照片、自定义壁纸和本机上传图标仍不会进入云端快照。

## English

Version 0.5.6 further hardens multi-device sync and public-product data controls. The data schema remains version 1; existing accounts, local data, and cloud snapshots are not rebuilt or cleared.

- Settings now merge per field. Widget flags, widget sizes, and calendar records merge at their nested entry level to reduce unrelated concurrent overwrites.
- Custom navigation page deletion now uses synchronized tombstones so an older offline device cannot resurrect a deleted page.
- Shortcut destinations are restricted to HTTP and HTTPS. Unsafe protocols from imports, complete backups, or historical cloud data are never opened.
- The Chrome and Edge extension uses the Chrome Search API and therefore respects the user's default search provider.
- Icon fallbacks now prioritize cache-friendly public providers and use a shorter failure timeout while retaining lazy loading and bounded caches.
- Signed-in users can permanently delete their account after re-entering the current email and password. Auth data, cloud snapshots, and the current device's account data are removed.
- The minimum sync-compatible client is now 0.5.6, preventing older whole-settings merge behavior from writing into hardened snapshots.
- Everyday installation instructions now point directly to the GitHub Release artifact.

This release does not add end-to-end encryption. Local photos, custom wallpapers, and uploaded local icons remain excluded from cloud snapshots.
