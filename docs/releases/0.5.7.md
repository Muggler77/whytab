# whytab 0.5.7

## 中文

0.5.7 是一次数据安全与恢复一致性修复版本，数据结构版本仍为 1。

- 新账号会话建立后，如果账号数据加载失败，客户端会清理该新会话并回到未登录空白数据，避免显示或上传旧账号状态。
- 撤销、同步回退、更新前自动备份恢复和完整备份导入会为恢复后的变更重写记录时间戳。
- 恢复操作中被移除的记录会保留删除标记，避免其他设备把旧记录重新合并回来。
- 最低可同步版本提升为 0.5.7，防止旧客户端使用过时的恢复与合并行为覆盖新快照。

## English

Version 0.5.7 hardens account switching and restore consistency while keeping data schema version 1.

- If loading a newly signed-in account fails, the client clears that new session and returns to an empty signed-out state instead of displaying or uploading the previous account state.
- Undo, sync restore, pre-update backup restore, and full backup import rewrite record timestamps for restored changes.
- Records missing from a restored snapshot receive deletion tombstones so another device cannot resurrect stale records.
- The minimum sync-compatible client is now 0.5.7 to prevent older restore and merge behavior from overwriting hardened snapshots.
