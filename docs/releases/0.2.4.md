# whytab 0.2.4

## 中文

0.2.4 修正桌面边缘导航的垂直位置，不改变数据结构、同步协议或账号数据。

### 修复内容

- 左侧或右侧桌面导航固定在视口边缘，并严格垂直居中。
- 始终显示和自动隐藏状态使用相同的垂直中心线。
- 完全隐藏后的边缘恢复按钮同步垂直居中。
- 移动端继续使用底部导航，不受桌面定位规则影响。
- 中间工作区保持对称边距，切换导航方向或显示方式时不发生横向位移。

### 数据安全

- 数据结构版本继续保持为 `1`。
- 不修改同步、登录、账号隔离或云端数据逻辑。
- 不执行用户数据迁移、删除、覆盖或重建。

## English

Version 0.2.4 corrects the vertical position of desktop edge navigation. It does not change the data schema, sync protocol, or account data.

### Fixes

- Keeps left or right desktop navigation pinned to the viewport edge and vertically centered.
- Uses one vertical center line for always-visible and auto-hide modes.
- Vertically centers the recovery control shown when navigation is hidden.
- Keeps the mobile bottom navigation unaffected by desktop positioning rules.
- Preserves symmetric workspace gutters with no horizontal shift when placement or visibility changes.

### Data Safety

- The data schema remains at version `1`.
- Sync, authentication, account isolation, and cloud-data behavior are unchanged.
- No user-data migration, deletion, overwrite, or rebuild is performed.
