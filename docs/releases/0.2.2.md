# whytab 0.2.2

## 中文

0.2.2 聚焦桌面导航、小组件排序、尺寸设置和整体视觉一致性。此次更新不改变 whytab 的数据结构与同步协议，现有快捷方式、文件夹、待办、笔记、倒计时、照片、页面、布局和账号数据会继续保留。

### 主要变化

- 桌面导航从页面中部移到屏幕边缘顶部，减少视觉漂浮感。
- 新增导航位置设置：左侧或右侧。
- 新增导航显示设置：始终显示、自动隐藏或隐藏；隐藏后仍保留窄边缘恢复按钮。
- 小组件排序改用 dnd-kit，支持鼠标、触屏和键盘，并提供独立拖动手柄和拖动预览。
- 右键菜单直接显示紧凑、标准、展开三种小组件比例预览，修改后主页立即更新。
- 主页和网站页图标统一为相同尺寸、圆角和图片留白；现有图标尺寸滑杆同时控制两页。
- 重做 12 个小组件的卡片层级、配色、标题、内容区域、加载态和空状态。
- 天气加载中会显示明确状态；倒计时突出核心日期；待办增加完成进度，数据较少时不再只剩大块空白。
- 拖拽模块改为按需加载，普通打开新标签页时不会加载低频编辑代码。
- 修复触摸设备媒体规则影响桌面图标尺寸，以及手机底部导航被桌面左右定位带偏的问题。

### 数据安全

- 数据结构版本继续保持为 `1`。
- 不执行删除、重建或覆盖用户数据的迁移。
- 新增的导航偏好只扩展现有设置对象；旧数据默认使用左侧、始终显示。
- 本地数据仍保存在当前浏览器的 IndexedDB；登录后的同步隔离与 Row Level Security 规则不变。

### 使用

- 在线版：打开 `https://why-tool.com/`，部署完成后刷新即可使用 0.2.2。
- Chrome / Edge：从本 Release 下载 `whytab-0.2.2-chrome-web-store.zip`，解压后在扩展管理页选择“加载已解压的扩展程序”。
- 已加载本地目录的测试用户：更新目录内容后，在扩展管理页点击“重新加载”。

## English

Version 0.2.2 refines desktop navigation, widget sorting, widget sizing, and visual consistency. It keeps the existing whytab data schema and sync protocol unchanged, so shortcuts, folders, todos, notes, countdowns, photos, pages, layouts, and account data remain compatible.

### Highlights

- Anchors desktop navigation near the screen edge instead of vertically centering it.
- Adds left/right navigation placement and always-visible, auto-hide, or hidden display modes.
- Keeps a narrow recovery trigger when desktop navigation is hidden.
- Replaces native HTML drag behavior with dnd-kit sorting for mouse, touch, and keyboard input.
- Adds a dedicated widget drag handle, drag preview, and immediate order persistence.
- Adds visual compact, standard, and expanded size choices directly to the widget context menu.
- Uses one icon size, radius, and image-padding system on both Home and Websites.
- Redesigns all 12 widgets with clearer hierarchy, distinct accents, stronger loading states, and useful sparse-data layouts.
- Lazy-loads the drag engine only when layout editing is enabled.
- Fixes touch media rules affecting desktop icon geometry and desktop side rules shifting the mobile bottom navigation.

### Data Safety

- The data schema remains at version `1`.
- This release performs no destructive user-data migration.
- New navigation preferences extend the existing settings object; legacy data defaults to always-visible navigation on the left.
- Local IndexedDB storage, account isolation, sync behavior, and Row Level Security remain unchanged.

### Install

- Web app: open `https://why-tool.com/` and refresh after deployment completes.
- Chrome / Edge: download `whytab-0.2.2-chrome-web-store.zip` from this Release, extract it, enable Developer mode, choose **Load unpacked**, and select the extracted folder.
- Existing unpacked installation: replace the local build directory with the 0.2.2 files and click **Reload** on the browser extensions page.
