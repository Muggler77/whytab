# whytab 0.5.1

## 中文

0.5.1 将官方网页入口迁移到免费的 Cloudflare Pages 地址 `https://whytab.pages.dev/`。本次变更只替换公开访问地址、登录回调、验证邮件链接、版本检查和部署配置，不迁移或重建 Supabase 数据库。

### 域名与部署

- 新官方网页地址为 `https://whytab.pages.dev/`。
- 移除旧自定义域名的 CNAME、扩展权限、硬编码回调和当前文档引用。
- 增加根目录 `wrangler.toml`，Cloudflare Pages 直接发布 `extension/dist`。
- GitHub Pages 继续作为无自定义域名的备用部署，不再声明旧 CNAME。

### 账号与数据

- Supabase 项目、用户账号、Row Level Security、同步表和数据结构保持不变。
- 已登录并成功同步的数据可在新地址登录同一账号后继续使用。
- Chrome / Edge 扩展使用自己的扩展存储空间，不受网页域名变化影响。
- 浏览器会按网址隔离 IndexedDB；旧网址下未登录且未同步的本地数据无法被新网址自动读取，需要通过 whytab JSON 导出和导入迁移。
- 新网址会建立新的登录会话，用户可能需要重新登录。

### 邮件限制

- `pages.dev` 是 Cloudflare 的共享域名，不能配置项目自己的 DKIM、SPF 或 `no-reply@whytab.pages.dev`。
- 免费过渡阶段使用 Supabase 内置 Auth 发件服务。
- 后续购买自有域名时，可以接入 Resend 或其他 Custom SMTP；数据库和用户数据无需迁移。

## English

Version 0.5.1 moves the official web entry point to the free Cloudflare Pages URL `https://whytab.pages.dev/`. This release changes public URLs, Auth redirects, verification-email links, update checks, and deployment configuration without migrating or rebuilding the Supabase database.

### Domain and deployment

- The new official web URL is `https://whytab.pages.dev/`.
- The retired custom-domain CNAME, extension permissions, hard-coded redirects, and current documentation references are removed.
- A root `wrangler.toml` configures Cloudflare Pages to publish `extension/dist`.
- GitHub Pages remains a fallback deployment without the retired custom CNAME.

### Accounts and data

- The Supabase project, user accounts, Row Level Security, sync tables, and data schema remain unchanged.
- Data that was signed in and successfully synchronized remains available after signing in with the same account at the new URL.
- Chrome and Edge extensions use their own extension storage origin and are unaffected by the web-domain change.
- Browsers isolate IndexedDB by origin. Local-only data that was never signed in or synchronized at the retired URL cannot be read automatically by the new URL and must be moved through a whytab JSON export and import.
- The new URL creates a new web session, so users may need to sign in again.

### Email limitation

- `pages.dev` is a shared Cloudflare domain and cannot provide project-owned DKIM, SPF, or `no-reply@whytab.pages.dev` records.
- The free transition uses Supabase's built-in Auth email sender.
- A future owned domain can use Resend or another Custom SMTP provider without migrating the database or user data.
